### Простая ментальная модель LVM (для тех, кто помнит DOS и разделы)

Раньше всё было просто.

Есть диск.
На диске есть разделы.
Каждый раздел — это **непрерывный кусок диска**.

```
|---sda1---|------sda2------|--sda3--|
```

Файловая система создаётся прямо на разделе.

Если удалить раздел — появляется свободное место между другими разделами.

---

# Как думает LVM

LVM меняет одну фундаментальную вещь:

**диск перестаёт делиться на большие куски.
Он делится на много маленьких одинаковых блоков.**

Эти блоки называются **PE (Physical Extents)**.

Упрощённо диск выглядит так:

```
|PE|PE|PE|PE|PE|PE|PE|PE|PE|PE|
```

Обычно один PE ≈ 4 MB.

---

# Где находятся PE

PE **физически лежат внутри PV**.

**PV (Physical Volume)** — это просто диск или раздел, который подготовили для LVM:

```
pvcreate /dev/sda
```

> Тут важно понимать: 
>
> Понятие "разделы" существует, как и раньше! И раздел - это более "низкоуровневая" сущность, чем LVM с ее структурой.
> ОБЫЧНО разделы использовать ВРЕДНО, лучше всего создавать PV прямо поверх физического диска.
> Делить PV, грубо говоря, на PV1 и PV2 на одном физическом диске смысла особого нет. Обычно весь диск делаеют как один PV, несколько дисков объеденяют в группу (назвается VG - virtual group),
> а потом уже пилят на "логические куски" (назыется LV logical volume)

После этого диск выглядит так:

```
PV /dev/sda

|PE|PE|PE|PE|PE|PE|PE|PE|
```

То есть:

```
Disk / Partition
        ↓
       PV
        ↓
|PE|PE|PE|PE|PE|PE|
```

---

# Что такое Volume Group (VG)

VG — это **логическая группа PV**.

Она **не содержит данные физически**.
Она просто **ведёт список всех PE, которые есть в PV**.

Пример:

```
pvcreate /dev/sda
pvcreate /dev/sdb
pvcreate /dev/sdc

vgcreate vg0 /dev/sda /dev/sdb /dev/sdc
```

Теперь VG видит один большой пул блоков:

```
VG vg0

|PE|PE|PE|PE|PE|PE|PE|PE|PE|PE|PE|PE|
```

Но важно:

эти PE **по-прежнему лежат на своих дисках**.

VG просто знает **где какие блоки находятся**.

---

# Что такое Logical Volume (LV)

LV — это **набор PE из VG**.

Когда создаётся LV:

```
lvcreate -L 50G -n home vg0
```

LVM берёт нужное количество PE из пула.

Например:

```
LV home

|PE|PE|PE|PE|PE|
```

Эти блоки могут лежать на разных дисках:

```
disk A: PE1
disk B: PE8
disk A: PE3
disk C: PE12
disk B: PE5
```

Но для системы это выглядит как **один обычный диск**.

---

# Файловая система

Теперь на LV создаётся файловая система:

```
mkfs.ext4 /dev/vg0/home
```

И монтируется как обычный раздел.

Файловая система **не знает**, где физически лежат PE.
Этим занимается LVM.

---

# Итоговая схема

```
Disk
  ↓
Partition (необязательно)
  ↓
PV  ← диск подготовлен для LVM
  ↓
PE  ← маленькие блоки внутри PV
  ↓
VG  ← объединяет PE из всех PV
  ↓
LV  ← набор PE из VG
  ↓
Filesystem
```
